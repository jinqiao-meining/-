SYSTEM OVERVIEW

一套最严谨的研究工作流

平台按“立项、收数、清洗、识别、估计、作图、写作、审阅、归档”九个阶段组织工作。 每个阶段都对应明确的输入、输出和人工确认点。

01

研究设计

定义问题、变量、样本与识别思路，形成项目章程。

02

数据治理

上传原始数据，生成字典、缺失报告、异常值报告和清洗脚本。

03

计量分析

执行描述统计、基准模型、稳健性、异质性和安慰剂检验。

04

写作审阅

生成草稿骨架、图表说明和诚信审阅意见，保留人工定稿。

AGENT SYSTEM

六大代理分工

每个代理只做自己最擅长的环节，由总控代理负责调度与汇总。

总控 Agent

接收研究问题，拆解任务，安排顺序，追踪进度，生成最终摘要。

数据治理 Agent

上传、检验、清洗、建字典、出质量报告，保留原始数据与分析数据分层。

计量识别 Agent

围绕 OLS、DID、IV、RDD、面板和时间序列提出规范化识别路径。

可视化 Agent

同时输出论文版和汇报版图表，自动附图注与一句话结论。

写作 Agent

生成引言、文献综述、方法和结果解释骨架，但不替代人工定稿。

诚信审阅 Agent

进行相似性线索审阅、AI 痕迹风险审阅与原创性增强建议。

ACCESSIBILITY

适老化与教师友好设计

大字与高对比

默认使用较大字号和高对比色彩，关键操作始终固定在一屏内。

一步一事

每个阶段只保留一个主按钮，减少复杂菜单和多层跳转。

结果附解释

每张图、每个模型结果旁边都要附一句话解释，避免只看到数字。

可导出

所有图表、工作单和研究日志都预留一键导出 PDF/PNG 的位置。

INTEGRITY REVIEW

学术诚信审阅模块

该模块不提供规避检测器的手段，只做相似性线索检查、AI 痕迹风险审阅和原创性增强建议。

相似性审阅

引导使用 CNKI 或 iThenticate 检查可能的高重合段落，并人工核验引用与改写质量。

AI 痕迹风险

识别模板化、空泛表述、无证据论断和缺少作者判断的段落，提示人工改进。

原创性增强

建议补入变量定义、样本处理、识别假设、局限性与真实研究判断，而不是机械同义替换。

严谨原则： AI 检测结果只能作为线索，不能作为定案；中文学术文本更应坚持人工复核与来源核验。
